- Maintain consistent structure

## ğŸ“¦ Pull Request Guidelines

### PR Title Format

Use one of these prefixes:
- `feat:` - New feature
- `fix:` - Bug fix
- `docs:` - Documentation changes
- `style:` - Code style changes (formatting, etc.)
- `refactor:` - Code refactoring
- `perf:` - Performance improvements
- `test:` - Adding tests
- `chore:` - Maintenance tasks

Examples:
- `feat: Add blog section`
- `fix: Correct mobile menu alignment`
- `docs: Update README with new deployment options`

### PR Description

Include:
- Description of changes
- Motivation and context
- Screenshots (for UI changes)
- Testing performed
- Related issues

### PR Checklist

- [ ] Code follows project style guidelines
- [ ] Changes are tested on multiple devices/browsers
- [ ] HTML is valid
- [ ] No console errors
- [ ] Documentation is updated (if needed)
- [ ] Commits are descriptive and atomic

When reporting bugs, please include:

2. **Steps to Reproduce**: Detailed steps to reproduce the issue
3. **Expected Behavior**: What you expected to happen
4. **Actual Behavior**: What actually happened
5. **Screenshots**: If applicable
6. **Environment**: Browser, OS, screen size
7. **Additional Context**: Any other relevant information
